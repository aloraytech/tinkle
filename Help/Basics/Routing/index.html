Routing

All Tinkle routes are defined in your route files, which are located in the router directory. These files are automatically loaded by your application.

The routers/web.php file defines routes that are for your web interface. For most applications, you will begin by defining routes in your routers/web.php file. The routes defined in routers/web.php may be accessed by entering the defined route's URL in your browser. For example, you may access the following route by navigating to http://yourdomain.com/user in your browser: 

use Tinkle\Router;
Router::get('user/update/{id}',[\App\Controllers\UserController::class,'update']);
Router::get('user',[\App\Controllers\UserController::class,'index']);
Router::get('greeting', function () {
    echo 'Hello World';
});
use Tinkle\Request;
Router::get('greet', [function (Request $request) {
    echo $request->getMethod();
},\Tinkle\Tinkle::$app->request]);
Router::get('greet/ing/{id}', [function ($id) {
    echo 'Hello World'. $id;
}]);
Router::get('greet/test/{id}', [function (\App\Controllers\TestController $id) {
    echo $id->getId();
},\App\Controllers\TestController::class]);

You can also group your routers for common web routes like 

Router::group('admin')->get('pages/update/{id}',[\App\Controllers\PageController::class,'item']);
Router::group('admin')->post('posts/{author}/{id}',[\App\Controllers\PostController::class,'item']);

The router allows you to register routes that respond to any HTTP method: 

Router::get($uri, $callback);
Router::post($uri, $callback);
Router::put($uri, $callback);
Router::delete($uri, $callback);

Sometimes you need to use wildcard routes for your application 

Router::any($uri, $callback);

Many Time You want to redirect routes from old URL to new URL 

Router::redirect('old/url','new/url',301); //Redirect

     
Router::redirect('old/url','new/url',302); //Permanent Redirect
